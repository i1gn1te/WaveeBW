2) Spotify routes - backend/src/routes/spotify.js

  • GET /api/spotify/search - szukanie piosenek.
  • GET /api/spotify/track/:trackId - szczegoly utworu + audio features.
  • GET /api/spotify/audio-features/:trackId - tylko audio features.
  • GET /api/spotify/top/tracks i /top/artists - top usera.
  • GET /api/spotify/genres - lista gatunkow.

Dla usera demo backend nie pyta Spotify i zwraca dane z mockData.js.

3) Reviews routes - backend/src/routes/reviews.js

Najmocniejsza rzecz: recenzja idzie jako upsert, czyli jak ten user juz recenzowal ten track, to idzie update, a nie drugi rekord. 

await prisma.review.upsert({
  where: { userId_trackId: { userId: req.userId, trackId } },
  update: { rating, content },
  create: { userId: req.userId, trackId, rating, content }
})
  • GET /api/reviews/my - recenzje usera z paginacja.
  • GET /api/reviews/track/:trackId - feed recenzji utworu.
  • DELETE /api/reviews/:reviewId - tylko wlasciciel moze usunac.
