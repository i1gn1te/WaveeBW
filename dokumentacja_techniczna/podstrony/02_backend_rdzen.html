02. Backend rdzen i middleware

Tutaj jest analiza glownego startu serwera oraz auth middleware.

Powrot do strony glownej

Plik: backend/src/index.js

To jest centralny punkt backendu. On robi 6 krytycznych rzeczy:

  1. Laduje konfiguracje ze zmiennych srodowiskowych.
  2. Uruchamia CORS i whitelist originow.
  3. Uruchamia parser JSON i obsluge cookie.
  4. Podpina route auth, user, reviews, spotify, playlists, recommendations.
  5. Daje endpoint zdrowia /api/health.
  6. Uruchamia globalny handler bledow.
app.use('/api/auth', authRoutes)
app.use('/api/user', userRoutes)
app.use('/api/reviews', reviewsRoutes)
app.use('/api/spotify', spotifyRoutes)
app.use('/api/playlists', playlistsRoutes)
app.use('/api/recommendations', recommendationsRoutes)

Plik: backend/src/middleware/auth.js

Ten middleware pilnuje, zeby request mial wazna sesje. Najpierw bierze token, potem go sprawdza.

const token = req.cookies.token || req.headers.authorization?.split(' ')[1]
if (!token) return res.status(401).json({ error: 'Authentication required' })

const decoded = jwt.verify(token, process.env.JWT_SECRET)
req.userId = decoded.userId
req.spotifyAccessToken = decoded.spotifyAccessToken

Co sie dzieje gdy JWT wygasnie

  • Middleware probuje odczytac token mimo expiracji.
  • Jesli ma refresh token Spotify, robi odswiezenie access tokena.
  • Wystawia nowe cookie JWT i request idzie dalej bez wyrzucania usera.

Plik: backend/src/lib/prisma.js

Singleton Prisma. Dzieki temu przy hot-reload nie tworzy sie duzo klientow DB i nie ma glupich problemow z polaczeniami. 

const globalForPrisma = globalThis
export const prisma = globalForPrisma.prisma ?? new PrismaClient()
if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma

Plik: backend/src/lib/spotify.js (rdzen integracji)

To jest najwiekszy plik logiczny backendu. Odpowiada za:

  • zbudowanie URL logowania Spotify,
  • wymiane code na tokeny,
  • refresh tokena,
  • search i track details,
  • audio features z fallbackiem na null,
  • rekomendacje z nowa strategia (bez starych deprecated endpointow).

Najwazniejsze: projekt juz jest przygotowany na zmiany Spotify po 2024, wiec nie opiera sie na wycofanym endpoint /recommendations w starej formie.
