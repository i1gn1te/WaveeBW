07. Uruchomienie, bezpieczenstwo i testy

Finalna sekcja: jak odpalic projekt, jak go testowac i na co uwazac.

Powrot do strony glownej

1) Uruchomienie lokalne

Backend

cd backend
npm install
npm run db:generate
npm run db:push
npm run dev

Frontend

cd frontend
npm install
npm run dev

Zmienne .env (backend)

PORT=3001
CLIENT_URL=http://localhost:5173
DATABASE_URL=postgresql://USER:PASS@localhost:5432/waveeProjectBW
JWT_SECRET=twoj_super_sekret
SPOTIFY_CLIENT_ID=...
SPOTIFY_CLIENT_SECRET=...
SPOTIFY_REDIRECT_URI=http://localhost:3001/api/auth/callback

2) Checklista testow recznych

  1. Login demo i wejscie do dashboard.
  2. Wyszukanie tracka i wejscie do szczegolow.
  3. Dodanie recenzji 4.5 i sprawdzenie listy recenzji.
  4. Generator playlisty po gatunku i po tracku.
  5. Usuniecie recenzji i weryfikacja odswiezenia danych.
  6. Wylogowanie i proba wejscia na strone chroniona.

3) Bezpieczenstwo - co jest i co warto dopchnac

Juz jest dobrze

  • JWT i middleware auth.
  • Walidacja danych przez Zod.
  • Kontrola wlasciciela zasobu (playlista/recenzja).
  • Obsluga bledow Spotify z czytelnym komunikatem.

Warto dorobic w kolejnym kroku

  • Rate limiting na endpointy logowania i search.
  • secure: true dla cookie na produkcji.
  • Testy automatyczne backend route.
  • Monitoring i centralny logger.

4) Co powiedziec przy obronie projektu

  1. Pokazac tryb demo jako zabezpieczenie prezentacji.
  2. Pokazac upsert recenzji i unikalnosc danych.
  3. Pokazac interceptor 401 i auto-refresh sesji.
  4. Pokazac jak frontend i backend dzialaja oddzielnie ale spojnie.

Taki flow zwykle robi bardzo dobre wrazenie, bo pokazuje nie tylko UI ale tez ogarnieta logike.
