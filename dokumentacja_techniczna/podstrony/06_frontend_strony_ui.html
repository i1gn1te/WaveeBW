06. Frontend strony i UI

Dokladny opis ekranow i komponentow, co user klika i co sie dzieje pod spodem.

Powrot do strony glownej

Home i Login

  • pages/Home.tsx ma przycisk Spotify i przycisk Demo.
  • pages/Login.tsx odpytuje backend o URL logowania Spotify.

W obu ekranach jest fallback demo, wiec prezentacja projektu nie pada przez brak API key.

Dashboard

pages/Dashboard.tsx odpytuje:

  • top tracks,
  • top artists,
  • statystyki recenzji,
  • ostatnie recenzje usera.

Ekran ma przelacznik zakresu czasu, np. 4 tygodnie, 6 miesiecy, all-time.

Search i Track

Search

  • formularz szukania po frazie,
  • wyniki jako karty trackow,
  • obsluga loading i error.

Track

  • szczegoly utworu (artist, album, czas),
  • audio features,
  • formularz recenzji (gwiazdki + komentarz),
  • lista recenzji i podobne utwory.

Reviews, Generator, Profile, Community

Reviews

Lista recenzji usera + usuwanie.

PlaylistGenerator

Dwa tryby: po gatunku albo po wybranym tracku. Ma usuwanie niepasujacych utworow i podmiane z puli zapasowej.

Profile

Edycja displayName, bio i favoriteGenres + statystyki.

Community

Szukanie userow i podglad profilu publicznego. W demo jest blokada tej funkcji.

Komponenty techniczne UI

  • AudioFeaturesDisplay.tsx - paski energii, danceability itd.
  • StarRating.tsx - gwiazdki z polowkami.
  • TrackCard.tsx - karta utworu.
  • index.css - globalny styl i scrollbar.
