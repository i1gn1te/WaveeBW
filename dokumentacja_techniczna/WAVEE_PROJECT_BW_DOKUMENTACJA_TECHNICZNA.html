waveeProjectBW - pelna dokumentacja techniczna

Wersja HTML do wygodnego czytania i szybkiego wydruku do PDF.

1. O czym jest projekt

waveeProjectBW to aplikacja muzyczna typu full-stack. Uzytkownik moze wyszukiwac piosenki, dodawac recenzje, sprawdzac audio features, budowac playlisty i przegladac spolecznosc. Aplikacja ma tez tryb demo, wiec dziala nawet bez pelnej konfiguracji Spotify.

2. Architektura

  • Frontend: React + Vite + Tailwind
  • Backend: Node.js + Express + Prisma
  • Baza: PostgreSQL
  • Integracja zewnetrzna: Spotify Web API

Przeplyw danych

  1. Frontend wysyla zapytanie pod /api/*.
  2. Backend obsluguje auth i logike domenowa.
  3. Backend bierze dane z PostgreSQL i/lub Spotify API.
  4. Zwrot JSON do frontendu.

3. Backend - technicznie

Serwer startuje z pliku backend/src/index.js, ustawia CORS, parser JSON, cookie parser i podpina route'y.

app.use('/api/auth', authRoutes)
app.use('/api/user', userRoutes)
app.use('/api/reviews', reviewsRoutes)
app.use('/api/spotify', spotifyRoutes)
app.use('/api/playlists', playlistsRoutes)
app.use('/api/recommendations', recommendationsRoutes)

Middleware auth bierze JWT z cookie, dekoduje go i gdy trzeba odswieza token Spotify.

const token = req.cookies.token || req.headers.authorization?.split(' ')[1]
const decoded = jwt.verify(token, process.env.JWT_SECRET)
req.userId = decoded.userId
req.spotifyAccessToken = decoded.spotifyAccessToken

4. Baza danych (Prisma)

Modele glowne:

  • User - dane konta i preferencje muzyczne
  • Review - recenzja + snapshot danych utworu
  • Playlist i PlaylistTrack - playlisty usera
  • LikedTrack - baza pod przyszle like/favorite

Wazny constraint: @@unique([userId, trackId]) w recenzjach.

6. Frontend - co i jak

Routing jest w frontend/src/App.tsx. Strony w protected route to m.in. Dashboard, Search, Track, Reviews, Generator, Community, Profile.

W frontend/src/lib/api.ts jest interceptor 401 i auto refresh sesji, co jest bardzo wygodne i uzytkownik mniej widzi bledy logowania.

if (status === 401 && !originalRequest._retry) {
  originalRequest._retry = true
  await api.post('/auth/refresh')
  return api(originalRequest)
}

7. Tryb demo

Tryb demo opiera sie o backend/src/lib/mockData.js. Pozwala pokazac prawie cala aplikacje bez API key Spotify. To jestesmy bardzo mocna rzecz na prezentacje.

8. Bezpieczenstwo i uwagi

  • Jest auth middleware + kontrole dostepu do zasobow usera.
  • Jest walidacja inputu przez Zod.
  • Jest fallback bledow Spotify do czytelnych komunikatow.
  • Do produkcji warto dac secure cookie i rate limiting.
